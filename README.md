## Overview

PitchForge AI is a full-stack SaaS workspace that turns a raw startup concept into a complete, structured business pitch in seconds. Founders, students, incubators, and hackathon teams enter three inputs — startup idea, industry, and revenue model — and receive a polished investor package that includes:

- A full narrative pitch (problem, solution, business model, market opportunity)
- A four-dimension startup score (Innovation · Market Demand · Scalability · Investor Appeal)
- A SWOT analysis
- A competitor landscape overview
- A seed-to-series valuation range estimate
- A downloadable PDF report

All analysis is generated by Google Gemini with deterministic fallbacks, so the application never crashes even without an API key.

## Features

| Feature | Description |
|---|---|
| **AI Pitch Builder** | Generates structured pitch sections from plain-language input via Gemini 1.5 Flash |
| **Startup Score Engine** | Scores four investor-relevant dimensions on a 0–100 scale |
| **SWOT Analysis** | Strengths, weaknesses, opportunities, and threats tailored to the startup |
| **Competitor Analysis** | Three positioned competitors with strengths, weaknesses, and market insights |
| **Valuation Estimator** | Low / medium / high pre-seed to seed-stage valuation ranges with reasoning |
| **PDF Export** | ReportLab-powered PDF download with all sections and the score table |
| **Pitch History** | Persisted history list with links back to any previous pitch report |
| **Graceful Fallbacks** | All AI methods fall back to deterministic, coherent data when Gemini is unavailable |
| **In-Memory Mode** | Works fully without MongoDB configured — ideal for local demos |
| **Rate Limiting** | IP-based sliding-window rate limiter built into the FastAPI middleware stack |

## Getting Started

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [MongoDB Atlas](https://mongodb.com/atlas) cluster (free tier works fine)
- A [Google AI Studio](https://aistudio.google.com) API key for Gemini

### 1 — Clone the repository

```bash
git clone https://github.com/your-username/pitchforge-ai.git
cd pitchforge-ai
```

### 2 — Backend setup

```bash
cd backend

# Create and activate a virtual environment
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
cp .env.example .env
# Edit .env and fill in MONGODB_URI, GEMINI_API_KEY, and SECRET_KEY
```

### 3 — Frontend setup

```bash
cd frontend

npm install

# Configure environment variables
cp .env.example .env.local
# Edit .env.local — set VITE_API_URL=http://127.0.0.1:8000
```

## Environment Variables

### Backend (`backend/.env`)

| Variable | Required | Description |
|---|---|---|
| `GEMINI_API_KEY` | Yes* | Google Gemini API key. Fallbacks activate if absent. |
| `GEMINI_MODEL` | No | Model name. Default: `gemini-1.5-flash` |
| `MONGODB_URI` | Yes* | MongoDB Atlas connection string. In-memory mode if absent. |
| `DATABASE_NAME` | No | Database name. Default: `pitchforge` |
| `SECRET_KEY` | **Yes** | JWT signing secret. Generate with `python -c "import secrets; print(secrets.token_urlsafe(32))"` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | No | JWT TTL in minutes. Default: `30` |
| `FRONTEND_URLS` | No | Comma-separated allowed CORS origins |
| `ALLOWED_HOSTS` | No | Comma-separated trusted hosts |
| `RATE_LIMIT_REQUESTS` | No | Max requests per window per IP. Default: `80` |
| `RATE_LIMIT_WINDOW_SECONDS` | No | Rate limit window in seconds. Default: `60` |
| `PORT` | No | Server port. Default: `8000` |

> *The app runs without MongoDB and without a Gemini key — all features use graceful fallbacks.

### Frontend (`frontend/.env.local`)

| Variable | Description |
|---|---|
| `VITE_API_URL` | Backend base URL, e.g. `http://127.0.0.1:8000`. The `/api` prefix is appended by the client automatically. |

## Running Locally

### Backend

```bash
cd backend
# With virtual environment activated:
python main.py
# Or directly with uvicorn:
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

API is available at `http://127.0.0.1:8000`  
Swagger UI at `http://127.0.0.1:8000/docs`

### Frontend

```bash
cd frontend
npm run dev
```

App is available at `http://localhost:5173`

## API Documentation

Interactive Swagger docs are available at `/docs` when the backend is running.

### Core Endpoints

#### `POST /api/generate-pitch`
Generate a complete AI pitch from three inputs.

**Request body:**
```json
{
  "startup_idea": "AI platform for clinic appointment management",
  "industry": "Healthcare SaaS",
  "revenue_model": "Monthly subscription per clinic"
}
```

**Response:** `201 Created`
```json
{
  "success": true,
  "pitch_id": "64f3a...",
  "data": {
    "startup_name": "CliniqPilot",
    "elevator_pitch": "...",
    "problem_statement": "...",
    "solution": "...",
    "business_model": "...",
    "revenue_streams": ["..."],
    "market_opportunity": "...",
    "investor_summary": "..."
  }
}
```

#### `POST /api/generate-score`
Score an existing pitch on four dimensions.

```json
{ "pitch_id": "64f3a..." }
```

#### `POST /api/generate-swot`
Generate SWOT analysis for an existing pitch.

```json
{ "pitch_id": "64f3a..." }
```

#### `POST /api/generate-competitors`
Generate competitor analysis for an existing pitch.

```json
{ "pitch_id": "64f3a..." }
```

#### `POST /api/generate-valuation`
Estimate a valuation range for an existing pitch.

```json
{ "pitch_id": "64f3a..." }
```

#### `GET /api/pitch/{pitch_id}`
Retrieve the full pitch document including all enrichment data.

#### `GET /api/history?limit=10&skip=0`
Return the most recent pitches (newest first).

#### `POST /api/export-pdf`
Stream a PDF report file for download.

```json
{ "pitch_id": "64f3a..." }
```

#### `POST /auth/signup`
Register a new user account.

```json
{
  "name": "Jane Founder",
  "email": "jane@example.com",
  "password": "securepassword"
}
```

#### `POST /auth/login`
Authenticate and receive a JWT token.

```json
{
  "email": "jane@example.com",
  "password": "securepassword"
}
```

#### `GET /auth/profile`
Get the authenticated user's profile.  
Requires: `Authorization: Bearer <token>`

## Deployment

### Backend — Render / Railway

1. Set the root directory to `backend/`
2. Build command: `pip install -r requirements.txt`
3. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
4. Add all environment variables from `backend/.env.example`
5. The `Procfile` is already configured for these platforms

### Frontend — Vercel

1. Set the root directory to `frontend/`
2. Build command: `npm run build`
3. Output directory: `dist`
4. Add `VITE_API_URL` pointing to your deployed backend URL
5. The `vercel.json` already includes SPA rewrite rules for client-side routing
